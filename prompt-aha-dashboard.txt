* Next.js (App Router)

* Tailwind CSS
* daisyUI

The system must be scalable, modular, mobile-friendly and follow modern SaaS best practices.

There are **two companies sharing the same system**:

* African Home Adventure
* Enchoro Wildlife Camp

# 🏢 PHASE 3 — MULTI-TENANT DATA MODEL

The system must support **multiple companies using the same database**.

### Add a Company model:

```ts id="s8dj2k"
interface Company {
  id: string;
  name: "African Home Adventure" | "Enchoro Wildlife Camp";
}
```

### Update all core models to include `companyId`:

```ts id="f9k3ls"
interface Booking {
  id: string;
  companyId: string;
  customerName: string;
  email: string;
  phone: string;
  safariPackage: string;
  country: "Kenya" | "Tanzania";
  startDate: string;
  endDate: string;
  status: "pending" | "confirmed" | "cancelled";
  paymentStatus: "paid" | "partial" | "unpaid";
  totalAmount: number;
}
```

Apply the same `companyId` pattern to:

* Payments
* Expenses
* Revenue

### Rules:

* All queries must be filtered by selected company
* Switching company updates all visible data
* Persist selected company in global state (Zustand or React Context - pick the most preferred for this project)
